## 2. AI 接入阶段

- 最初网站只显示 DeepSeek。

- 后续加入流式回复、停止生成、重新生成、Markdown/GFM 和复制回复。

## 3. 登录与账户阶段

- 使用 Supabase 邮箱 Magic Link，无密码注册登录。
- 修复本地回调、生产回调和过期邮件链接问题。

## 4. 点数和计费阶段

- 新用户初始化测试点数。

- 增加点数流水、管理员充值和点数明细页面。

- 管理后台增加运营统计和系统状态检查。

## 5. 充值订单阶段

- 价格页提供体验包、标准包和进阶包。
- 测试阶段采用“创建订单后联系管理员人工收款”。
- 增加用户订单中心和管理员订单管理。
- 管理员确认到账时原子更新订单、余额和流水。
- 增加支付方式、流水号、管理员备注、订单取消和过期保护。
- 后续目标是接入正式在线支付，当前不要把人工订单误称为已接入支付。

## 6. 法务、支持与运维阶段

- 完成用户协议、隐私政策、退款说明和帮助反馈。
- 增加健康检查、生产诊断、业务一致性检查和错误恢复页面。

- 建立运维手册，记录 Supabase 暂停、登录、部署、数据库和 AI 故障处理。
- 在 2026-06-14 DNS 故障恢复后，增加 GitHub Actions 每日生产冒烟监控。

## 7. 移动端阶段

依次优化：

- 管理员订单、用户、点数管理、流水和运营概览
- 用户点数、账户、价格、支持和登录
- 首页导航和多模型介绍
- 聊天导航、历史会话、动态视口、中文输入法和手机发送按钮

## 8. 重要产品决策

1. **聊天错误对用户友好，对服务端保留细节。**
2. **扣点必须服务端原子完成，失败必须退款。**
3. **管理员身份来自环境变量，不硬编码。**
4. **人工充值必须通过订单确认函数完成，保证账务一致。**
5. **密钥、数据库数据和逐字用户对话不进入公开 GitHub。**
6. **每次功能修改后进行 TypeScript、ESLint 和 Git diff 检查。**
7. **每次稳定修改提交并推送 `main`，由 Vercel 自动部署。**
