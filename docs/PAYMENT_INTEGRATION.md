# 在线支付接入说明

更新时间：2026-06-15

## 当前状态

网站当前继续使用人工确认收款流程。代码库已经加入在线支付所需的数据库安全底座，但尚未连接微信支付、支付宝或其他支付服务商。

在正式接入任何支付平台前，不应把浏览器跳转成功、前端支付结果或用户提交的截图作为到账依据。用户加点只能由服务端验证过签名的支付回调触发。

## 已准备的安全能力

迁移文件：

`supabase/migrations/20260615_payment_webhook_safety.sql`

该迁移提供：

1. 支付平台订单号与交易流水号唯一约束；
2. `payment_webhook_events` 回调事件账本；
3. 按支付平台和事件 ID 去重；
4. 服务端金额一致性校验；
5. 订单行锁，防止并发重复加点；
6. 订单、用户余额、点数流水和回调状态的原子更新；
7. 仅保存回调摘要和必要标识，不保存完整敏感支付报文。

## 正式支付适配器必须遵守的顺序

1. 接收支付平台的原始 HTTPS 回调；
2. 使用支付平台官方方式验证签名和时间戳；
3. 解析平台事件 ID、商户订单号、平台订单号、交易流水号和实付金额；
4. 将原始报文计算为不可逆摘要，日志中不得记录私钥、API 密钥或完整付款人信息；
5. 仅在平台明确表示支付成功时调用 `complete_online_recharge_order`；
6. 根据函数返回值向支付平台返回成功或可重试响应；
7. 重复事件必须返回成功，但不得重复加点。

## 上线前需要准备

- 已认证的企业或个体工商户主体；
- 对应支付平台的商户账号与已开通的网页/移动端支付产品；
- 平台要求的商户证书、私钥、API 密钥或应用密钥；
- 可公开访问的 HTTPS 回调地址；
- 退款、对账、异常订单和密钥轮换流程；
- 沙箱或小额真实订单的完整验收记录。

## 下一步

确定首个支付渠道后，再完成：

1. 支付渠道服务端适配器；
2. 创建支付会话/二维码接口；
3. 签名回调接口；
4. 支付状态查询与前端轮询；
5. 管理后台回调事件和异常订单页面；
6. 退款接口与每日对账任务。

在支付适配器完成并通过验收前，价格页必须继续显示“人工确认收款”，不能宣称已支持在线支付。
